# Препринт для Overleaf

Готовый исходник статьи на LaTeX. Текст и числа совпадают с `paper/preprint.md`,
разметка переведена в формат для журнала.

## Состав

- `main.tex` - весь препринт: титул, аннотация, разделы, Таблицы 1-3, рисунки,
  список литературы.
- `figures/flow.pdf` - Рисунок 1, поток пациентов (вектор).
- `figures/schema.pdf` - схема этапов исследования (вектор).
- `figures/*.mmd` - исходники схем в формате Mermaid, на случай перерисовки.

Схемы векторные (PDF), не пикселят при увеличении.

## Как открыть в Overleaf

1. Заархивировать папку `overleaf` целиком в zip (вместе с подпапкой `figures`).
2. На overleaf.com: New Project -> Upload Project -> выбрать zip.
3. Menu -> Settings: Compiler = pdfLaTeX, Main document = `main.tex`.
4. Recompile. Кириллица собирается через T2A + babel, дополнительных шрифтов не нужно.

Все используемые пакеты (babel, booktabs, graphicx, rotating, hyperref, cmap)
входят в стандартный TeX Live на Overleaf, ставить ничего не надо.

## Что заполнить автору

В тексте оставлены пометки `[заполнить ...]`:

- титул: аффилиация, электронная почта автора для переписки, дата;
- раздел "Этическое одобрение" в методах и в "Дополнительных сведениях";
- финансирование, конфликт интересов, вклад авторов, благодарности;
- в списке литературы пометки `[уточнить ...]` для части источников (тома,
  страницы, инициалы).

## Перерисовать схемы (если правились данные)

Схемы отрисованы из `.mmd` через mermaid-cli. Команда (нужен Node и установленный
Chrome или Edge):

```
npx -y -p @mermaid-js/mermaid-cli mmdc -i figures/flow.mmd -o figures/flow.pdf -f -p pconfig.json
```

где `pconfig.json` указывает на браузер, например:

```
{ "executablePath": "C:\\Program Files\\Google\\Chrome\\Application\\chrome.exe", "args": ["--no-sandbox"] }
```
