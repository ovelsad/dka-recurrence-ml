# Клинические факторы рецидивирующего течения диабетического кетоацидоза

Научно-исследовательская работа: какие клинические показатели при поступлении
отличают рецидивирующее течение диабетического кетоацидоза (ДКА) от единичного
эпизода. Состав факторов получен двумя независимыми путями, классической
статистикой и методом SHAP поверх интерпретируемых моделей машинного обучения,
после чего результаты сопоставлены. Итог - рукопись статьи и дополнительно
прототип сервиса поддержки решений врача.

## О задаче и данных

Бинарная классификация: рецидивирующее течение против единичного эпизода ДКА.
Когорта 273 пациента одного центра с установленным исходом, доля рецидивирующего
течения 31.9%. Выборка малая и несбалансированная, поэтому акцент на честной
валидации, а не на завышенных метриках.

Важная особенность дизайна: исход и предикторы относятся к одной госпитализации,
времени до события и окна наблюдения в данных нет. По классификации TRIPOD модель
диагностическая, она распознает уже наступившее состояние, а не предсказывает
будущий эпизод. С моделями, предсказывающими госпитализацию за 30, 90 или 180
дней, результаты напрямую не сопоставимы.

Данные пациентов закрыты: сырой датасет, обработанные таблицы, обученные модели и
отчеты с идентификаторами не публикуются (см. `.gitignore`). Воспроизведение
требует исходного файла `data/raw/dka_bd_289.xlsx`, доступ к нему по обоснованному
запросу к авторам в рамках этических ограничений.

## Методология

- Без утечек данных: разбиение train/test один раз до предобработки, импутация,
  масштабирование и синтетика обучаются только внутри фолдов кросс-валидации,
  отложенный тест трогается один раз.
- Признаки для моделей отобраны только по взаимной коррелированности предикторов,
  без обращения к исходу, поэтому отбор не завышает оценку качества.
- Честная валидация при малой выборке: стратифицированная и вложенная
  кросс-валидация, доверительные интервалы метрик по бутстрэпу.
- Метрики под клинику и дисбаланс: ROC-AUC, PR-AUC, чувствительность,
  специфичность, калибровка, анализ кривых принятия решений.
- Статистика по фиксированному протоколу (выбор критериев по нормальности и
  дисперсиям, отношение шансов, поправка Бенджамини-Хохберга), подробности в
  `docs/methodology.md`.
- Отчетность по рекомендациям STROBE для наблюдательных исследований, для части,
  касающейся разработки моделей, дополнительно учтен TRIPOD+AI.

## Эксперименты и результаты

- Разделяющая способность ROC-AUC 0.71-0.78 по вложенной кросс-валидации и
  0.72-0.80 на отложенном тесте, доверительные интервалы широкие и перекрываются.
- Четыре модели на одном наборе из 25 признаков без мультиколлинеарности:
  логистическая регрессия, случайный лес, XGBoost, CatBoost. Попарный критерий
  DeLong не выявил ни одной значимой пары из 6 сравнений, сложные модели не
  превзошли логистическую регрессию. Единую модель не выделяем.
- Рейтинг моделей на отложенном тесте оказался обратным рейтингу на
  кросс-валидации, что при 55 пациентах в тесте ожидаемо и служит
  предостережением против выбора модели по одной отложенной выборке.
- Стекинг (0.7699 против 0.772 у лучшей одиночной модели), синтетическая
  балансировка (SMOTE и варианты) и преобразования признаков прироста не дали.
- Сырые вероятности откалиброваны лучше методов Платта и изотонической регрессии,
  заданное заранее правило применения калибровки не выполнилось ни для одной
  модели.
- Со значимым различием между группами после поправки Бенджамини-Хохберга
  выделены 10 признаков (Таблица 1 в рукописи).
- В топ-10 по SHAP одновременно у всех четырех моделей входят пять признаков: вид
  инсулинотерапии, суточная доза инсулина, HbA1c, ЛПВП и глюкоза при поступлении.
  Четыре из них значимы и при сравнении групп, пятый (глюкоза) выявлен только
  многомерным подходом. При бутстрэпе обучающей части общими для всех моделей
  стабильно остаются два признака (92% и 89% повторов), остальные три в 32-40%.
- Проверены две заранее заданные гипотезы. Клиническая: связь приема алкоголя с
  рецидивирующим течением подтверждена (ОШ 2.27, 95% ДИ 1.23-4.20, p=0.008).
  Методологическая: превосходство сложных моделей над логистической регрессией не
  продемонстрировано.

Полное описание с интервалами и таблицами - в рукописи `paper/manuscript_ru.md`
(русская версия) и `paper/manuscript_en.md` (английская). Собранные PDF в формате
Springer sn-jnl лежат рядом, исходники `manuscript_ru.tex` и `manuscript_en.tex`.

## Структура репозитория

- `src/` - пакет с кодом всех этапов: очистка, статистика, импутация, подготовка
  признаков, моделирование, тюнинг, калибровка и порог, SHAP, проверка
  устойчивости, ансамбли, проверка гипотез, модели сервиса.
- `notebooks/` - этапы 00-15 как тонкие слои над `src/`.
- `service/app.py` - прототип сервиса на Streamlit.
- `docs/methodology.md` - методология и статистический протокол.
- `paper/` - рукопись статьи в двух языках, markdown, LaTeX и PDF.
- `data/`, `reports/`, `models/` - не коммитятся (данные пациентов и производные
  артефакты), пересоздаются кодом.

## Воспроизведение

После размещения датасета в `data/raw/` модули запускаются как пакеты по порядку
этапов:

```
python -m src.cleaning        # очистка -> data/processed/dka_clean.parquet
python -m src.tables          # Таблица 1 по группам
python -m src.missingness     # анализ пропусков
python -m src.imputation      # стратегии импутации
python -m src.modeling        # сравнение стратегий (CV на train)
python -m src.optuna_tuning   # вложенная CV (Optuna)
python -m src.evaluation      # оценка на отложенном тесте
python -m src.threshold       # точка отсечения по индексу Юдена
python -m src.interpret       # SHAP
python -m src.shap_stability  # бутстрэп-проверка совпадения признаков
python -m src.bootstrap_ci    # бутстрэп-интервалы и OOB
python -m src.ensembles       # стекинг
python -m src.synthetic       # синтетическая балансировка
python -m src.hypothesis_test # проверка гипотезы (DeLong)
python -m src.service_model   # модели сервиса
```

Зерно случайности одно на проект (`RANDOM_SEED` в `src/config.py`). Ноутбуки
дублируют этапы для просмотра.

## Ограничения

Дизайн поперечный: исход установлен ретроспективно, на момент измерения
предикторов он в большинстве случаев уже реализовался, поэтому речь идет о связях,
а не о доказанной причинности и не о прогнозе.

Выборка одноцентровая и малая, внешней валидации нет. Часть лабораторных
показателей с высокой долей пропусков (до 58%). Список значимых признаков получен
на полной когорте и относится к разведочному анализу, он требует подтверждения на
независимой выборке. Совпадение признаков между моделями подтверждено бутстрэпом
лишь частично: два признака из пяти держатся в 92% и 89% повторов, остальные три в
32-40%.

Отдельная оговорка по признаку вида инсулинотерапии: доля пациентов без
инсулинотерапии составила 28.9% при единичном эпизоде против 1.1% при
рецидивирующем течении, то есть признак частично воспроизводит определение исхода
(дебют заболевания) и самостоятельным фактором риска считаться не может.

Точка отсечения выбрана по индексу Юдена и служит только для иллюстрации структуры
ошибок, клинической рекомендацией она не является.

## Прототип сервиса (дополнительно)

Вспомогательная демонстрация, не основной результат работы. Прототип на Streamlit
по показателям пациента оценивает вероятность того, что течение рецидивирующее,
относит наблюдение к группе по регулируемому порогу и поясняет вклад признаков
(коэффициенты для логрега, SHAP для деревьев). Все вычисления локальные, данные
наружу не передаются.

```
python -m streamlit run service/app.py
```
