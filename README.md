# Прогнозирование рецидива диабетического кетоацидоза

Научно-исследовательская работа: прогноз повторного эпизода диабетического
кетоацидоза (ДКА) по клиническим данным пациента интерпретируемыми моделями
машинного обучения. Итог - интерпретируемая прогностическая модель, научный
препринт и дополнительно прототип сервиса поддержки принятия решений врача.

## О задаче и данных

Бинарная классификация: рецидив против единичного эпизода ДКА. Когорта 273
пациента одного центра с известным исходом, доля рецидивов 31.9%. Выборка малая и
несбалансированная, поэтому акцент на честной валидации и клинической
применимости, а не на завышенных метриках.

Данные пациентов закрыты: сырой датасет, обработанные таблицы, обученные модели и
отчеты с идентификаторами не публикуются (см. `.gitignore`). Воспроизведение
требует исходного файла `data/raw/dka_bd_289.xlsx`, доступ к нему по обоснованному
запросу к авторам в рамках этических ограничений.

## Методология

- Без утечек данных: разбиение train/test один раз до предобработки, импутация,
  масштабирование, отбор признаков и синтетика обучаются только внутри фолдов
  кросс-валидации, отложенный тест трогается один раз.
- Честная валидация при малой выборке: стратифицированная и вложенная
  кросс-валидация, доверительные интервалы метрик по бутстрэпу.
- Метрики под клинику и дисбаланс: ROC-AUC, PR-AUC, чувствительность,
  специфичность, калибровка, анализ кривых принятия решений.
- Статистика по фиксированному протоколу (выбор критериев по нормальности и
  дисперсиям, отношение шансов, поправка Бенджамини-Хохберга), подробности в
  `docs/methodology.md`.
- Отчетность по стандарту TRIPOD-AI.

## Эксперименты и результаты

- Разделяющая способность ROC-AUC 0.72-0.80 по вложенной кросс-валидации и на
  отложенном тесте, доверительные интервалы широкие и перекрываются.
- Восемь моделей-финалистов: логистическая регрессия, случайный лес, XGBoost,
  CatBoost на двух наборах признаков. Попарный критерий DeLong не выявил значимых
  различий (0 из 28), сложные модели не превзошли логистическую регрессию. Единую
  модель не выделяем, в сервис вынесены все восемь.
- Стекинг, синтетическая балансировка (SMOTE и варианты) и преобразования
  признаков прироста качества не дали.
- Порог подбирали под целевую чувствительность не ниже 0.75: режим скрининга с
  высоким NPV (0.75-0.92) при умеренном PPV.
- Десять значимых предикторов согласуются между классической статистикой и SHAP:
  суточная доза инсулина, вид инсулинотерапии, длительность СД, HbA1c, ЛПВП, ХБП
  (стадия и альбуминурия), ретинопатия, полинейропатия, прием алкоголя.
- Проверены две заранее заданные гипотезы. Клиническая: связь приема алкоголя с
  рецидивом подтверждена (ОШ 2.27, p=0.008). Методологическая: превосходство
  сложных моделей над логистической регрессией отклонено.

Полное описание с интервалами и таблицами - в препринте `paper/preprint.md`.

## Структура репозитория

- `src/` - пакет с кодом всех этапов: очистка, статистика, импутация, подготовка
  признаков, моделирование, тюнинг, калибровка и порог, SHAP, ансамбли, проверка
  гипотез, модели сервиса.
- `notebooks/` - этапы 00-15 как тонкие слои над `src/`.
- `service/app.py` - прототип сервиса на Streamlit.
- `docs/methodology.md` - методология и статистический протокол.
- `paper/preprint.md` - научный препринт.
- `data/`, `reports/`, `models/` - не коммитятся (данные пациентов и производные
  артефакты), пересоздаются кодом.

## Воспроизведение

После размещения датасета в `data/raw/` модули запускаются как пакеты по порядку
этапов:

```
python -m src.cleaning        # очистка -> data/processed/dka_clean.parquet
python -m src.tables          # Таблица 1 по группам
python -m src.missingness     # анализ пропусков
python -m src.imputation      # стратегии импутации
python -m src.manual_impute   # ручная клиническая импутация
python -m src.modeling        # сравнение стратегий (CV на train)
python -m src.optuna_tuning   # вложенная CV (Optuna)
python -m src.evaluation      # оценка на отложенном тесте
python -m src.threshold       # подбор порога
python -m src.interpret       # SHAP
python -m src.bootstrap_ci    # бутстрэп-интервалы и OOB
python -m src.ensembles       # стекинг
python -m src.synthetic       # синтетическая балансировка
python -m src.hypothesis_test # проверка гипотезы (DeLong)
python -m src.service_model   # модели сервиса
```

Зерно случайности одно на проект (`RANDOM_SEED` в `src/config.py`). Ноутбуки
дублируют этапы для просмотра.

## Ограничения

Одноцентровая малая демонстрационная выборка, внешней валидации нет. Часть
лабораторных показателей с высокой долей пропусков (до 58%). Набор значимых
признаков отобран на полной когорте, поэтому его оценка может быть слегка
завышена. Прогноз - инструмент скрининга, а не замена клинического решения.

## Прототип сервиса (дополнительно)

Вспомогательная демонстрация, не основной результат работы. Прототип на Streamlit
по показателям пациента выдает калиброванную вероятность рецидива, относит к группе
риска по регулируемому порогу и поясняет вклад признаков (коэффициенты для логрега,
SHAP для деревьев). Все вычисления локальные, данные наружу не передаются.

```
python -m streamlit run service/app.py
```
